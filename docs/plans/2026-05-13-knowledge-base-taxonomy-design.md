# 知识库分类设计方案

## 概述

- **用途**: 教学/课程大纲
- **结构**: 两层分类（领域 + 子类）
- **数据层**: Domain → SubCategory → Chapter → Article

---

## 一、分类体系

### 顶层分类

按技术领域划分：

```
📂 后端开发
📂 前端开发
📂 AI / 机器学习
📂 基础设施
```

**设计原则**: 按领域分组而非语言，便于跨领域关联（如 Java + AI）

### 子类设计

#### 后端开发

```
📂 后端开发
├── Java
│   ├── Java基础
│   ├── Java高级（JVM、并发）
│   ├── 主流框架（Spring、MyBatis）
│   └── 微服务
├── Python
│   ├── Python基础
│   ├── Web开发（Django/Flask）
│   └── 数据处理
└── Go（未来扩展）
```

#### 前端开发

```
📂 前端开发
├── 基础（HTML/CSS/JS）
├── 框架（React/Vue）
└── 工程化
```

#### AI / 机器学习

```
📂 AI / 机器学习
├── 机器学习基础
├── 深度学习
├── NLP/大模型
└── AI工程化
```

#### 基础设施

```
📂 基础设施
├── 数据库
│   ├── 关系型（MySQL/PostgreSQL）
│   └── NoSQL（Redis/MongoDB）
├── 消息队列
│   ├── RabbitMQ
│   ├── Kafka
│   └── RocketMQ
└── 云原生
```

## 二、数据层设计

### TypeScript 类型

```ts
// 领域（顶层）
interface Domain {
  id: string;           // 'backend' | 'frontend' | 'ai' | 'infra'
  title: string;        // '后端开发'
  icon: string;          // '⚙️'
  color: string;         // 主题色
  subCategories: SubCategory[];
}

// 子类（第二层）
interface SubCategory {
  id: string;           // 'java' | 'python'
  title: string;        // 'Java'
  icon: string;          // '☕'
  domainId: string;      // 所属领域
  description: string;
  chapters: Chapter[];   // 该子类下的章节
}

// 章节（现有 Chapter 不变）
interface Chapter {
  id: string;
  title: string;
  icon: string;
  description: string;
  difficulty: number;
  color: string;
  articles: ArticleMeta[];
}
```

## 三、页面设计

### 4.1 整体布局

```
┌────────────┬──────────────────────────────────────┐
│            │                                      │
│  侧边栏    │  主内容区                             │
│  (全局导航) │  (随路由变化)                         │
│            │                                      │
└────────────┴──────────────────────────────────────┘
```

- **侧边栏**: 全局导航，所有页面共享
- **主内容区**: 主页 / 子类详情页 / 文章页

---

### 4.2 侧边栏

使用 **2级联动下拉框 + 动态目录**，侧边栏始终只展示2层深度：

```
┌─────────────────────────────┐
│  🦝 Raccoon                 │
│  Java AI 技术学习平台        │
├─────────────────────────────┤
│                             │
│  [⚙️ 后端开发 ▼]  [☕ Java ▼] │  ← 2级联动下拉框
│                             │
├─────────────────────────────┤
│                             │
│  📚 Java核心基础             │  ← Chapter级
│    ├ Java基础               │  ← Article级
│    └ Java高级               │
│                             │
│  📦 集合框架深度解析         │
│    ├ Collection体系架构      │
│    ├ List深度解析            │
│    ├ Set深度解析             │
│    └ HashMap深度剖析         │
│                             │
│  ⚡ 并发编程                 │
│    ├ 线程基础               │
│    └ 锁机制                 │
│                             │
├─────────────────────────────┤
│  🏠 主页                    │
└─────────────────────────────┘
```

**交互逻辑**:
- 选择领域下拉框 → 子类下拉框更新为该领域下的子类列表
- 选择子类下拉框 → 下方目录刷新为该子类下的 Chapter + Article
- 例如：选择 [AI] → 子类下拉框更新为 [LLM基础] [LangChain]
- 目录始终只展示2层：Chapter → Article
- 当前选中项高亮 + 左侧蓝色标记

**优点**:
- 侧边栏始终2层深度，不会过深
- 下拉框始终可见上下文，不会迷失
- 切换成本低于折叠/展开
- 移动端更友好

- 节点间用箭头表示学习顺序
- 阶段右侧显示预计学习时长
- 底部汇总：总专题数、总时长、总文章数
